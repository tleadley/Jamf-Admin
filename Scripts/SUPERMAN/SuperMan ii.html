MacOS Update Policy

Secure & Enforced macOS Updates with Jamf Pro

Key Policy Objectives

30

Day Grace Period

Updates are ignored for 30 days post-release for testing.

5

User Deferrals

Users can postpone the update 5 times.

24

Hour Deferral

Each deferral lasts for 24 hours.

45-Day Enforcement Window

This chart shows the breakdown of the update lifecycle, starting from the end of the 30-day grace period.

Update Enforcement Timeline (User Experience)

Day 0: OS Update Released

Apple releases a new macOS update. The `DaysDeadlineStart` key is activated.

Day 30: Grace Period Ends

The 30-day testing window is over. S.U.P.E.R.M.A.N. begins its workflow, downloads the update, and starts prompting the user with 5 deferrals.

Day 40: Soft Deadline

All 5 deferrals are used or 10 days have passed. The update dialog becomes more persistent and can't be timed out (`DialogTimeoutSoftDeadline`).

Day 45: Hard Deadline

The `HardDeadlineDays` is reached. The update is mandatory, and the device will be forced to restart and install the update.

Secure Jamf Pro Setup (API Client)

This 2-part process ensures S.U.P.E.R.M.A.N. authenticates securely using Jamf's modern API Client method, not an admin user account.

Step 1: Create API Role

Define *what* the script can do. Grant these minimum privileges:

  • Read Computers
  • Create/Read Managed Software Updates
  • Send Remote Command (Download and Install OS X Update)
→

Step 2: Create API Client

Create the credentials and link the Role. This gives you the Client ID & Secret.

  • Token Expiration: Set to **60 minutes** for best security.
  • Record ID & Secret: Securely copy the generated credentials.

Jamf Policy Deployment

Finally, deploy the script and securely pass the API credentials (ID and Secret) as encrypted parameters in the policy.
