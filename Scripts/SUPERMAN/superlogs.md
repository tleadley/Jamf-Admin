# 🧹 S.U.P.E.R.M.A.N. Log Archive Pruning

This policy ensures that the historical log archives generated by the S.U.P.E.R.M.A.N. (`super`) script do not consume excessive disk space on client Macs.

The `super` script automatically archives its logs into ZIP files, but it does **not** automatically delete these old archives. This policy deploys a configuration file for the macOS native log rotation service, **`newsyslog`**, to automatically prune these archives.

---

## ⚙️ Deployment Overview

The configuration is deployed via a standard Jamf Pro policy running a script as the root user.

|**Item**|**Details**|
|---|---|
|**Target Directory**|`/etc/newsyslog.d/` (The correct configuration directory for macOS).|
|**Configuration File**|`super_logs.conf`|
|**Pruning Rule**|Keep a maximum of **30** archived logs, and delete any log archive older than **90 days**.|
|**Jamf Frequency**|**Once per computer** (The file only needs to be deployed once).|

---

## 📜 Jamf Policy Script Content

The script ensures the configuration file is present and correctly formatted in the system's log management directory.

Bash

```
#!/bin/bash

# Target directory and file path for the newsyslog configuration
CONFIG_DIR="/etc/newsyslog.d"
CONFIG_FILE="${CONFIG_DIR}/super_logs.conf"

# The newsyslog configuration content:
# Fields: LogFile [Mode] [Count] [Size] [When] [Flags]
# (644 permissions, 30 max archives, check weekly, delete files older than 90 days)
NEWSYSLOG_CONFIG='/Library/Management/super/logs-archive/*.zip 644 30 * $W0T0 90Z'

# 1. Ensure the configuration directory exists
/bin/mkdir -p "$CONFIG_DIR"

# 2. Overwrite the existing configuration file with the new content
echo "$NEWSYSLOG_CONFIG" > "$CONFIG_FILE"

# 3. Set ownership and permissions (required for newsyslog)
/usr/sbin/chown root:wheel "$CONFIG_FILE"
/bin/chmod 644 "$CONFIG_FILE"

exit 0
```

---

## 📌 Verification

After the policy runs, you can confirm the configuration file is correctly installed on a client machine via Terminal:

Bash

```
ls -l /etc/newsyslog.d/super_logs.conf
```

The output should show the file with `root` ownership and `644` permissions. Log rotation (pruning) will occur automatically when macOS runs its daily maintenance routines.
